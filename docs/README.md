# Black Clover RP - Sandbox Documentation

This project contains the complete technical and functional documentation for the Black Clover RP server on Sandbox.

## Documentation Methodology (Visual Paradigm - Standard SDLC)

Each domain is organized as follows:
1. **01_Requirements**: Functional requirements, User Stories, and Use Cases.
2. **02_Analysis**: System behavior, State machines, and Sequence flows.
3. **03_Design**: Data schemas, API/Function definitions, and Component architecture.

## Domains

### 01. Core Engine & Systems
*   **Mana/Grimoire Mechanics**: Mana regeneration, Grimoire binding, Attribute generation.
*   **Player Attributes**: Physical stats, Magic stats, Affinity.

### 02. Magic & Combat
*   **Spells**: Mana cost, Cooldowns, Scaling.
*   **Elemental Interactions**: Fire > Wind, etc.
*   **Combat States**: Knockback, Stun, Magic Shield.

### 03. Progression & Economy
*   **Ranking**: Magic Knight exams, Promotions, Yul (Currency).
*   **Grimoires**: Tier systems, Rarity, Progression.

### 04. World & Environment
*   **Map Zones**: Forsaken, Common, Noble, Royal Realm.
*   **NPC & Questing**: Magic Knight captains, Dungeon rewards.

### 05. UI & UX
*   **HUD**: Mana bar, HP, Active spell.
*   **Menus**: Grimoire interface, Character stats.
